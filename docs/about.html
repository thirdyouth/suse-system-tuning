1. 关于¶

openSUSE Leap 可用于非常广泛的商业或者科技数据中心的应用场景。SUSE可以确保 openSUSE Leap 可以进行设置，以最佳的性能来适应不同的操作目的。然而，OpenSUSE Leap 在部署到大量运行服务器中时需要遇到各种不同的需求，例如文件系统服务器。

我们不可能发布一个对所有的工作负载进行了优化的版本。不同的工作负载在某些方面差异很大。这些方面包括 I/O 访问模式，内存访问模式，和进程调度。一种完美适配某种确定工作负载的配置可能降低对另一种工作负载的性能。例如，I/O敏感任务，如数据库请求处理，通常与CPU敏感任务，如视频编码，它们需要完全不同的需求。Linux 的通用型让它可以通过对您的系统进行配置在每个应用场景下都可以发挥最好的性能。

本手册向您介绍管理和分析系统的方法。它将描述这些管理系统资源和优化系统的方法。该指南将不会提供特殊场景下的应用，因为每个服务器都有它独有的需求。相反，它将会使你能够彻底的分析了解您的服务器，并且最大限度的使用它们。

  • 系统优化常规注意事项

    优化系统需要非常细致的全盘规划和实施。本章主要了解哪些步骤才可以成功的提升系统性能。

  • 系统监视

    Linux 提供了大量的工具来管理系统的各个方面。本章主要了解如何使用这些工具，以及如何读取和分析系统日志文件。

  • 内核监视

    Linux 内核本身提供方法来检测各个系统的大小部件。本章主要介绍 SystemTap，一种用于编写内核模块的脚本语言，可用于分析和过滤数据。通过使用内核探针，和 Perf 来收集调试信息和查找问题瓶颈。最后通过 Oprofile 来监视应用程序。

  • 资源管理

    了解如何设置裁剪系统来满足服务器需要。了解如何使用电源管理，同时确保系统性能等级满足当前需要。

  • 内核调优

    Linux 内核可以通过 sysctl 或者 /proc 文件系统进行优化。本章内容覆盖优化 I/O 性能和调节 Linux 调度进程方法。同时也会描述内存管理的基本理念，以及展示内存管理如何优化才能满足特定应用和使用模式的需要。进一步的，将会讲述如何优化网络性能。

  • 系统崩溃处理

    本章让您能够优化和处理应用或者系统崩溃。将会介绍跟踪工具，例如 strace 或者 Itrace，并且会讲述如何通过 Kexec 和 Kdump 来处理系统崩溃问题。

1.1. 可用文档¶

我们为编写的书籍提供了不同语言的翻译，以及HTML和PDF两种格式。关于 openSUSE 这一系统，我们为用户和系统管理员提供了如下手册：

  • 入门指南

    引导用户一步一步的使用 openSUSE DVD 或者 ISO 镜像进行系统安装，对 GNOME 和 KDE 桌面环境进行简单的介绍，包括主要程序的运行。同时对 LibreOffice 进行整体的概览，如何使用相关模块进行文档编写，表格处理，图像和演示文档的创建。

  • 参考手册

    对 openSUSE 进行一个整体的概括，包括高级系统管理任务，主要面向系统管理员和对基本系统管理有初步了解的家庭用户。该手册提供了有关高级部署方案，系统管理，关键系统组件交互，以及不同网络设置和 openSUSE 提供的文件服务等方面的详细信息。

  • 安全指南

    对系统安全的基本概念进行介绍，包括本地和网络安全相关内容。展示产品固有安全软件的使用，如 AppArmor （方便用户指定每个程序可以读，写，和执行的文件），或者收集所有安全相关事件的审计系统。

  • 系统分析和优化指南

    本指南是一个问题检测，解析和优化的系统管理员手册。主要内容是关于如何使用检测工具检查，和优化您的系统，以及如何有效的管理资源。同时也涵盖了常用问题，解决办法，以及附加帮助和文档资源的整体概括。

  • KVM 虚拟化

    本手册提供了 openSUSE 系统上 KVM （基于内核的虚拟机）虚拟化设置和管理的介绍。同时也展示了使用 libvirt 和 QEMU 进行虚拟机客户端管理的方法。

您可以在系统的 /usr/share/doc/manual 目录下或者桌面的帮助中心找到多数产品手册的 HTML 格式版本。您可以在 http://www.suse.com/documentation 找到最新更新的文档，您可以下载其中的 PDF 或者 HTML 格式的产品手册。

1.2. 问题反馈¶

下面是不同的问题反馈方式：

  • 问题和功能请求

    您可以使用 https://bugzilla.novell.com/ 来进行产品组件的问题反馈，和功能请求。对于文档中的问题，请向相应产品的 文档 计划提交反馈。

    如果您对 Bugzilla 不熟悉，可以查看下面的文档：

    • http://en.opensuse.org/openSUSE:Submitting_bug_reports
    • http://en.opensuse.org/openSUSE:Bug_reporting_FAQ

  • 用户评价

    我们希望听到您对我们的手册和产品中包含的其他文档的评价和建议。请利用在线文档的每个页面底部的用户评论功能，或者打开 http://www.suse.com/documentation/feedback.html 然后添加您的评论。

  • 邮件

    您也可以发送邮件到 doc-team@suse.de 对产品的文档进行反馈。请确保在邮件中包含文档标题，产品版本，以及文档的发布日期。如果是有关问题反馈和功能方面的建议，请对问题有一个详细的描述，并引用相关的页面（或者 URL）和章节。

1.4. 手册生成¶

对于本文档，由 Novdoc ，即 DocBook （）的一个子集编写。XML 源文件使用 xmllint 校对，并由 xslproc 进行处理，最后使用定制的 Norman Walsh 表格样式转换成 XSL-FO。最终的 PDF 格式由 XEP 通过 RenderX 进行渲染。用于构建本手册的开源工具和环境可以使用 openSUSE 提供的 daps 包进行安装。该项目的主页位于 http://daps.sf.net/ 。

注解

对于本文档，即本翻译手册，由 reStructuredText 编写，使用 sphinx 和 make 进行管理和生成网页和 PDF 格式文件。
